README (ENGLISH)

Predictive system for a specific heart condition, such as Paroxysmal Atrial Fibrillation. Various technologies were applied, ranging from data analysis and pre-processing to deployment and implementation.

Technologies used included: cloud computing, Docker, Grafana, Python, MQTT broker, Google Colab, and Fuzzy Logic, among others.

The analysis and design stage, which consists of the following, can be observed:

A. DATABASE CHARACTERIZATION: The data extracted from electrocardiograms is analyzed, and the most relevant aspects are identified.
B. BIOMARKER SELECTION PROCESS: The markers that vary during an episode of Atrial Fibrillation are detected, and the signal is processed to obtain them.
C. PREDICTIVE ALGORITHM: The predictive model is defined, and based on this, the membership function and critical points are also established.
D. SENSOR AND IoT INFRASTRUCTURE SIMULATOR: The communications infrastructure is defined.
E. SYSTEM DEPLOYMENT: It is implemented locally for testing.
F. CLOUD COMPUTING IMPLEMENTATION: The cloud platform, service model, and, where applicable (in this case, IaaS), VM characteristics are selected. The final deployment is also performed.

In the #5.0 society in which we live, where #ArtificialIntelligence is increasingly integrated into our lives, I believe the potential applications of #bigdata and #IoT through a #microservicesinfrastructure in the #cloud are very interesting.

LEAME (ESPAÑOL)

Sistema predictivo para una enfermedad cardíaca específica, como lo es la Fibrilación Auricular Paroxística. Para ello se han aplicado distintas tecnologías, que abarcaron desde el análisis y pre-procesamiento de los datos hasta el despliegue e implementación del mismo.

Se utilizaron tecnologías como: #cloudcomputing, #docker, #grafana, #python, #brokerMQTT, #googlecolab y #fuzzylogic, entre otras.  

Se podrá apreciar la etapa de análisis y diseño que consta de:
 
A. CARACTERIZACIÓN BBDD: se analizan los datos extraídos de los electrocardiogramas y se ven los aspectos más relevantes que estos tienen.
B. PROCESO DE SELECCIÓN DE BIOMARCADORES: se detectan los marcadores que varían durante un episodio de Fibrilación Auricular, y se trata la señal para obtenerlos.    
C. ALGORITMO PREDICTOR: se define el modelo predictor, y en base a este también, la función de membresía y los puntos críticos.
D. SIMULADOR DE SENSOR E INFRAESTRUCTURA IOT: se define la infraestructura de comunicaciones.
E. DESPLIEGUE DEL SISTEMA: se implementa de manera local para hacer las pruebas pertinentes.
F. IMPLEMENTACIÓN EN CLOUD COMPUTING: se selecciona la plataforma de cloud, modelo de servicios, y según corresponda, es este caso IAAS, características de la VM. Además del correspondiente despliegue final.
